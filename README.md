# Свадебное приглашение — Максим и Мария

Одностраничный адаптивный сайт-приглашение на свадьбу 29 августа 2026 года.
Чистый HTML/CSS/JavaScript, без сборщиков и зависимостей.

## Структура проекта

```
index.html
styles.css
script.js
assets/
  location.jpg               — фото банкетного зала (используется на сайте)
  dress-code-reference.jpg   — референс палитры дресс-кода (НЕ выводится на сайте)
media/
  silk-background.mp4        — фоновое видео стартового экрана
  wedding-music.mp3          — фоновая музыка
README.md
```

## Стартовый экран, видео и музыка

- Стартовый экран (`#intro` в `index.html`) блокирует прокрутку основного сайта,
  пока гость не нажмёт «Открыть приглашение». По клику: запускается музыка с
  плавным нарастанием громкости, стартовый экран плавно исчезает, прокрутка
  разблокируется, фокус переходит на `<main>`.
- Состояние «приглашение уже открыто» запоминается в `sessionStorage` на время
  вкладки — при обновлении страницы стартовый экран не показывается повторно.
  Управляется константой в начале `script.js`:
  ```js
  const REMEMBER_OPENED_INVITATION = true; // false — показывать стартовый экран при каждой перезагрузке
  ```
- Видео `media/silk-background.mp4` подключено как постоянный фоновый слой
  (`autoplay muted loop playsinline`), с CSS-фолбэком (`.silk-bg__fallback`,
  тёплый градиент) и остановкой воспроизведения при `prefers-reduced-motion: reduce`.
- Музыка `media/wedding-music.mp3` никогда не запускается автоматически до
  клика пользователя (требование браузеров + явное условие задачи). Кнопка
  управления музыкой — `#music-toggle` в правом нижнем углу, с SVG-иконками
  «вкл/выкл», `aria-pressed` и сохранением выбора в `sessionStorage`.
- Чтобы заменить видео или музыку, положите файлы с теми же именами в
  `media/`, либо поменяйте пути в `<source src="...">` (видео) и
  `<audio src="...">` в `index.html`.

## Как запустить локально

Сайт не требует сборки. Достаточно открыть `index.html` в браузере — двойным
щелчком или через "Open with → Browser".

Для корректной работы `fetch`-запросов формы (если вы подключите
`FORM_ENDPOINT`) рекомендуется открывать сайт через локальный сервер, а не
напрямую с диска. Например, любой из вариантов:

```bash
# Python 3
python -m http.server 8000

# Node.js (npx)
npx serve .
```

Затем откройте `http://localhost:8000`.

## Где что менять

### Имена и дата свадьбы
- Имена и дата на первом экране: `index.html`, секция `.hero`
  (`.hero__name` × 2, `.hero__date-month/-day/-year`).
- Дата в подвале сайта: `index.html`, блок `<footer class="site-footer">`.
- Дата и время для таймера обратного отсчёта: `script.js`, константа
  `WEDDING_DATE_UTC_MS` в начале файла (сейчас `29.08.2026 12:30` по
  московскому времени, UTC+3).

### Программа дня
`index.html`, секция `<section class="section" id="program">` —
список `<ol class="timeline">`. Каждый пункт — это `<li class="timeline__item">`
с временем и названием слева (`.timeline__time`, `.timeline__label`) и
описанием справа (`.timeline__descr`).

### Цвета дресс-кода
`index.html`, секция `<section class="section" id="dresscode" ...>`, список
`<ul class="palette">`. Каждый образец — `<span class="palette__swatch" style="--swatch:#XXXXXX">`.
Меняйте HEX-код в атрибуте `style`, порядок и форму менять не нужно.
Общая палитра сайта (фон, акценты, текст) задана CSS-переменными в начале
файла `styles.css`, в блоке `:root`.

### Фотография локации
Файл `assets/location.jpg`. Чтобы заменить фото, положите новый файл с тем же
именем в папку `assets/` (либо измените путь в атрибуте `src` у изображения
внутри `.card__photo` в `index.html`, секция `#location`). Изображение
кадрируется через `object-fit: cover` и `object-position` в `styles.css`
(селектор `.card__photo img`) — при необходимости поправьте `object-position`,
если важная часть здания обрезается на новом фото.

### Ссылка на локацию (Яндекс Карты)
`index.html`, секция `#location`, атрибут `href` у кнопки «На карте».

## Анкета гостя: где включить реальную отправку ответов

Вся логика анкеты находится в `script.js`. В самом начале файла — переменная:

```js
const FORM_ENDPOINT = "";
```

Пока она пустая, анкета работает в **тестовом режиме**: при отправке форма
проверяется, но данные не уходят никуда во внешний сервис — они сохраняются
локально в `localStorage` браузера (ключ `wedding-rsvp-responses`), и
показывается сообщение:

> «Анкета сохранена в тестовом режиме. Для получения ответов подключите обработчик формы».

Чтобы получать реальные ответы, впишите в `FORM_ENDPOINT` URL вашего
обработчика — тогда форма начнёт отправлять POST-запрос с JSON:

```json
{
  "guestName": "Иван Иванов",
  "attendance": "yes",
  "joining": ["registry", "banquet"],
  "drinks": ["whiteWine", "sparkling"],
  "comment": "Спасибо за приглашение!",
  "submittedAt": "2026-07-19T10:00:00.000Z"
}
```

Ниже — три способа подключить бесплатный/недорогой обработчик.

### 1. Formspree

1. Зарегистрируйтесь на [formspree.io](https://formspree.io) и создайте новую форму.
2. Скопируйте выданный endpoint вида `https://formspree.io/f/xxxxxxxx`.
3. Вставьте его в `FORM_ENDPOINT` в `script.js`:
   ```js
   const FORM_ENDPOINT = "https://formspree.io/f/xxxxxxxx";
   ```
4. В настройках формы на Formspree включите приём JSON-запросов (Formspree
   принимает `Content-Type: application/json` по умолчанию для fetch-запросов).
5. Ответы будут приходить на почту, привязанную к аккаунту Formspree, и
   сохраняться в личном кабинете.

### 2. Google Apps Script (Google Таблицы)

1. Создайте новую Google Таблицу для ответов.
2. Откройте `Расширения → Apps Script` и вставьте примерно такой код:
   ```js
   function doPost(e) {
     const data = JSON.parse(e.postData.contents);
     const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
     sheet.appendRow([
       data.submittedAt,
       data.guestName,
       data.attendance,
       (data.joining || []).join(', '),
       (data.drinks || []).join(', '),
       data.comment
     ]);
     return ContentService.createTextOutput(JSON.stringify({ ok: true }))
       .setMimeType(ContentService.MimeType.JSON);
   }
   ```
3. Разверните скрипт: `Развернуть → Новое развёртывание → Веб-приложение`.
   Доступ — «Все, у кого есть ссылка».
4. Скопируйте URL веб-приложения (заканчивается на `/exec`) и вставьте его в
   `FORM_ENDPOINT` в `script.js`.

### 3. Make (ранее Integromat)

1. Создайте новый сценарий в [make.com](https://www.make.com) с триггером
   **Webhooks → Custom webhook**.
2. Скопируйте URL вебхука и вставьте его в `FORM_ENDPOINT` в `script.js`.
3. Добавьте в сценарий следующий модуль (например, «Google Sheets → Add a
   Row» или «Email → Send an email»), чтобы обрабатывать входящие данные
   анкеты (`guestName`, `attendance`, `joining`, `drinks`, `comment`,
   `submittedAt`).
4. Включите сценарий (`Scheduling → ON`).

### 4. Telegram-бот (рекомендуется, если ответы нужны в Telegram)

Готовый файл-релей уже лежит в проекте: `telegram-relay.gs`.

1. Откройте [script.google.com](https://script.google.com) → «Новый проект»,
   замените содержимое на файл `telegram-relay.gs` из корня этого репозитория.
2. Слева — значок шестерёнки «Настройки проекта» → «Свойства скрипта» →
   добавьте два свойства:
   - `TELEGRAM_BOT_TOKEN` — токен вашего бота от [@BotFather](https://t.me/BotFather);
   - `TELEGRAM_CHAT_ID` — id чата, куда слать анкеты.
3. «Развернуть» → «Новое развёртывание» → тип «Веб-приложение».
   Выполнять от имени: «Меня». Доступ: «Все».
4. Скопируйте выданный URL (заканчивается на `/exec`) и вставьте его в
   `FORM_ENDPOINT` в `script.js`.

**Почему не напрямую из браузера.** Токен бота — это учётные данные: если
вставить его прямо в `script.js`, он окажется в исходном коде страницы и
будет виден любому посетителю сайта (вкладка «Просмотр кода», вкладка
«Сеть» в инструментах разработчика). Это позволит кому угодно слать
сообщения от имени вашего бота. Apps Script решает это: код и «Свойства
скрипта» доступны только владельцу проекта, наружу отдаётся только
безопасный URL для приёма POST-запросов с сайта — токен через него никогда
не передаётся и не виден.

### Замена тестового сохранения на реальную отправку

Как только `FORM_ENDPOINT` заполнен, скрипт (`script.js`, обработчик
`form.addEventListener('submit', ...)`) автоматически переключается с
`saveToLocalStorage(payload)` на `submitToEndpoint(payload)` — правки в
разметке или дополнительный код не требуются.

## Открытые вопросы, которые требуют подтверждения клиента

- **Фотография для соцсетей (Open Graph).** Сейчас в `og:image` используется
  `assets/location.jpg` как временное решение. Замените на отдельное
  изображение 1200×630, если для соцсетей нужна другая картинка.
- **Favicon.** В `index.html` стоит заглушка (`<link rel="icon" href="data:,">`).
  Добавьте свой файл (например, `assets/favicon.ico` или `.svg`) и обновите
  ссылку.
- **Шрифты.** Подключены локально через `@font-face` в `styles.css` (без
  внешних сервисов, файлы лежат в `media/`): «Prata» — строгие заголовки,
  «Azbuka03_D» — каллиграфические акценты и декоративные надписи,
  «Montserrat» — текст, формы и кнопки. Роли заданы CSS-переменными
  `--font-display`, `--font-script`, `--font-body` в `:root` (`styles.css`).
- **Реальный обработчик анкеты** (Formspree / Apps Script / Make) — нужно
  выбрать и подключить один из вариантов выше, иначе ответы гостей будут
  сохраняться только локально в браузере каждого гостя.
